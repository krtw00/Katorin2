# リファクタリング: $ARGUMENTS

以下の原則に従ってリファクタリングを実施してください：

## リファクタリング原則

1. **動作を変更しない** - 既存の機能を保持
2. **小さなステップで進める** - 一度に大きく変えない
3. **テストを維持** - 各ステップでテストがパスすることを確認

## チェックポイント

### コード整理
- [ ] 重複コードの抽出
- [ ] 長いメソッドの分割
- [ ] 複雑な条件文の簡略化
- [ ] マジックナンバーの定数化

### 設計改善
- [ ] 単一責任原則の適用
- [ ] 依存性注入の活用
- [ ] インターフェースの抽出
- [ ] 不要な抽象化の削除

### 可読性向上
- [ ] 意味のある命名
- [ ] 適切なコメント
- [ ] 一貫したフォーマット

## 出力形式

```
## 変更前の問題点
[現状の課題]

## 変更内容
[具体的な変更と理由]

## 変更後の利点
[改善された点]
```
