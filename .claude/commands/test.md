# テスト: $ARGUMENTS

## テスト作成方針

### 1. テストケースの設計
- 正常系: 期待通りの入力での動作
- 異常系: 不正な入力でのエラーハンドリング
- 境界値: エッジケースの動作
- 統合: コンポーネント間の連携

### 2. テスト構造（AAA パターン）
```
// Arrange: 準備
// Act: 実行
// Assert: 検証
```

### 3. テストの命名規則
```
[対象]_[条件]_[期待結果]
例: createUser_withValidData_returnsUser
```

## 実行コマンド

```bash
# 全テスト実行
npm test

# 特定ファイルのみ
npm test -- [ファイル名]

# カバレッジ付き
npm test -- --coverage
```

## 出力形式

```
## 追加したテストケース
- [テスト名]: [説明]

## カバレッジ
- 対象ファイル: [カバレッジ%]

## 注意点
- [考慮すべき事項]
```
