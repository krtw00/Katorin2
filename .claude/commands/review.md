# コードレビュー

以下の観点でコードをレビューしてください：

## チェック項目

### 1. コード品質
- 命名規則は適切か
- 関数・メソッドは単一責任を守っているか
- 重複コードはないか
- 適切なコメントがあるか

### 2. セキュリティ
- 入力値の検証は適切か
- SQLインジェクション対策
- XSS対策
- 認証・認可の実装

### 3. パフォーマンス
- N+1クエリはないか
- 不要な処理はないか
- キャッシュの活用

### 4. 保守性
- テストは書かれているか
- エラーハンドリングは適切か
- 依存関係は適切か

## 出力形式

問題を優先度別に分類：
- **Critical**: 必ず修正
- **Warning**: 修正推奨
- **Suggestion**: 検討事項

具体的な修正例を含めてください。
