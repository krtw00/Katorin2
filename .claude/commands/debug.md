# デバッグ: $ARGUMENTS

以下の手順でデバッグを実施してください：

## 1. エラー情報の収集
- エラーメッセージとスタックトレースを確認
- 再現手順を特定
- 関連するログを調査

## 2. 根本原因の分析
- 失敗箇所を特定
- 関連コードを調査
- 依存関係を確認

## 3. 修正の実装
- 最小限の修正を提案
- 影響範囲を評価
- テストケースを追加

## 4. 検証
- 修正後の動作確認
- 既存テストが通ることを確認
- エッジケースの考慮

## 出力形式

```
## 根本原因
[原因の説明と証拠]

## 修正内容
[具体的なコード変更]

## テスト方法
[検証手順]

## 再発防止策
[今後の対策]
```
