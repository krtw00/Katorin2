# Roadmap

## Rebuild First

実装前提は `現行 Next.js アプリの延命` ではなく、`Rails 8 ベースの再構築` とする。

先にやること:

- 新しい技術基盤を確定する
- Rails アプリを立ち上げる
- Supabase Postgres へ接続する最小構成を作る
- WMGP Phase 1 に必要な migration を最小セットで切る

この再構築が終わるまでは、現行アプリへの機能追加を止める。

## Phase 1

目標は `WMGP Season 8 を運営専用台帳として最後まで回せること`。

必須:

- ログイン必須の非公開運用
- 薄い案内トップ
- ホームの運営タスク表示
- リーグ中心の情報設計
- チームと参加者の手入力
- Week 作成
- 対戦カード手入力
- 対戦日時管理
- Judge名、ルームID、観戦ID の後追い入力
- ラウンド、卓単位の結果入力
- 自動集計
- 1対戦ごとの結果画像出力
- 決勝進出と決勝台帳の管理

後回し:

- 順位表画像の高度化
- 対戦カード自動生成
- チーム向け入力
- 高度な監査ログ

## Phase 1.5

WMGP の実運用を通して、運営 UI を詰める段階。

候補:

- ホームの優先タスク最適化
- 入力フォーム短縮
- 修正導線の改善
- 画像出力オペレーション短縮

## Phase 2

目標は `ジャッジ負担軽減のためのチーム結果報告`。

前提:

- Discord でのオーダー、デッキ提出は継続
- Katorin2 が受け持つのは試合結果報告

導入するもの:

- チームごとの自チーム入力
- ラウンドごとの結果報告
- 両チーム報告の照合
- 食い違い箇所の可視化
- ジャッジによる差分確認と公式確定

理想形:

- 正しい入力ならチーム報告だけで完結する
- ジャッジは矛盾や裁定が必要な箇所だけ対応する

## 将来

WMGP 以外でも使うために、次を順次差し替え可能にする。

- 順位計算ルール
- 進行ルール
- 決勝進出ロジック
- 画像レイアウト

ただし、共通化は Phase 1 の成立より優先しない。まず WMGP の現場で回ることが先である。
