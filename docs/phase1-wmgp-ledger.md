# Phase 1: WMGP Ledger (完了)

## 目的

Katorin2 を `WMGP Season 8 を運営だけで最後まで回せる戦績管理台帳` として作り直す。

主目的: 既存の Discord 中心運用を壊さずに、 運営が Excel で持っていた情報を Web UI に置き換え、 入力 / 修正 / 集計 / 画像出力の負担を減らす。

実装・運用ともに完走済み。 本 doc は 「Phase 1 で何を達成したか」 の正本として残す。

## 前提

- ログイン必須
- 利用者は共通の運営アカウント
- 参加者向け公開画面は作らない
- 参加者への共有は画像出力を基本とする
- チャット / 掲示板 / チーム向け自己管理 UI は Phase 1 では扱わない
- アプリケーション基盤は `Rails 8 + Hotwire`
- DB は `Cloud SQL (PostgreSQL 17)`

## スコープ内 (実装済み)

- 複数リーグを管理できるアカウント構造
- `League` 配下でのチーム / 参加者 / 画像出力管理
- `League > Phase` 配下でのブロック / Week / 対戦管理
- 運営による手入力前提の台帳 UI
- Week 作成
- 対戦カードの手動作成
- 対戦日時の管理
- 試合結果の詳細入力
- 集計と順位計算
- 1対戦ごとの結果画像出力
- 順位表画像出力 (StandingsExports)
- 決勝進出管理と決勝系台帳の運用
- ダッシュボード (進行中試合ハイライト含む)
- 初回セットアップ導線 (organizer_setup)

## スコープ外 (Phase 1 では扱わない)

- 単発大会
- 参加者向けポータル
- チームによるオーダー提出 UI
- チームによるデッキ提出 UI
- Discord の置き換え
- 公開向けマーケティング LP

これらは Phase 1 のスコープ外として固定する。 必要になった時点で Plane Issue として再起票し、 Phase 3 (= 他リーグ / 大規模大会展開) のスコープと併せて再評価する。

## 運用フロー

Phase 1 の基本フロー:

1. Week を作成する
2. 対戦カードを手動で作成する
3. 対戦ごとに日時を設定する
4. 試合結果を入力する
5. 結果画像を作成する
6. 必要に応じて順位表と決勝進出を更新する

補足:

- `Phase` が予選と決勝を分ける
- `Week` は週番号のみを持つ
- 日付と時刻は `対戦` が持つ
- `Judge名` / `ルームID` / `観戦ID` は対戦作成時ではなく後追い入力

## 情報設計

画面は `ホーム` と `台帳一覧` を分離する。

### ホーム

ホームは一覧ページではなく、 運営のリアルタイム作業キューを出す。

表示要素:

- 今日の対戦
- 進行中試合のハイライト
- ルーム情報未入力
- 結果未入力
- 画像未出力
- 要確認の試合

### リーグ配下

リーグを最上位オブジェクトとし、 下位オブジェクトをすべて内包する。

- `League`
- `League > Phases`
- `League > Participants`
- `League > Teams`
- `League > Phase > Blocks`
- `League > Phase > Weeks`
- `League > Phase > Matches`
- `League > Exports`

`Teams` や `Participants` はトップレベル機能にしない。

## 認証

運営専用アプリなので、 認証は最小構成。

- 運営だけがログインできる
- 共通運営アカウント運用を許容する
- 参加者やチーム向け権限は持たない

## 画像出力

最優先は `1 対戦ごとの結果画像`。

要件:

- WMGP の現行結果画像と実質同一の UI を出せる
- 正本デザインは `/home/iguchi/Downloads/WMGP/image1.png`
- デッキ名 / 並び順 / W/L 表示 / Judge / Week 表記 / 日付 / 総合スコアを含める

順位表画像 (StandingsExports) も Phase 1 のスコープに含めて実装済み。

## ルート URL

`https://katorin2.codenica.dev/` は宣伝 LP ではなく、 関係者向けの薄い案内トップ。

含めるもの:

- Katorin2 が WMGP 運営用台帳である説明
- 関係者向けシステムである旨
- ログイン導線
- 現在運用中リーグへの導線

含めないもの:

- 公開向けの長い説明
- 参加者向け機能一覧
- 外部集客目的のコピー
