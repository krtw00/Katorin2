## Product Role

WMGP 向け PoC のアプリは、チーム側が触る大会ポータルではなく、運営専用の台帳として扱う。

- 連絡、日程交渉、通常コミュニケーションは Discord に残す
- アプリは試合運営に必要な情報入力、集計、記録、出力に集中する
- チーム側の学習コストを増やす機能は初期段階では持たない
- WMGP で実運用しながら改修する前提で、他大会対応は後続に回す

## Scope For PoC

Discord ログとルールブックから見た最小スコープ:

- チームとロスターの台帳管理
- 週次の対戦カード管理
- 日程確定の管理
- オーダー提出の管理
- ラウンド結果入力
- 試合結果確定と順位計算
- 運営向けの結果一覧と通知用情報出力

初期スコープから外すもの:

- アプリ内チャット
- チーム側セルフ操作
- 個人向け公開ポータル
- OAuth を前提にした個人認証
- 他大会向けの汎用化

## Weekly Flow

### 1. 週開始前

- チーム登録状態を確認する
- メンバー追加、削除、移籍申請を反映する
- 日曜 23:59 締切を超えた変更は翌週扱いにする

### 2. 月曜

- 対戦カードを作成、確定する
- ブロック戦またはトーナメントの対戦表を運営画面で確認できるようにする
- 通知文面の元データを出せるようにする

### 3. 火曜から金曜 23:59

- 両チーム代表者の Discord 上の調整結果を、運営がアプリへ記録する
- 試合日時、対戦ルーム担当、備考を保持する
- 未確定試合を一覧で追えるようにする
- 金曜 23:59 までに未確定の試合を把握できるようにする

### 4. 試合当日 2 時間前まで

- オーダー提出を受け付ける
- 各ラウンドに出すメイン 3 名とサブ 1 名を記録する
- デッキ提出状況も同じ試合単位で記録する

### 5. 試合中

- ラウンドごとの勝敗を入力する
- 個人マッチ単位の結果を入力する
- 50 分制限、切断、警告、没収などの裁定メモを残せるようにする
- 次ラウンドのオーダー変更可否を判定できるようにする

### 6. 試合後

- 試合結果を確定する
- 勝ち点、得失点差、順位表を自動更新する
- リザルト表や共有用の出力に使える状態へまとめる

## Core Entities

PoC で必要な最小単位:

- `OrganizerAccount`
  - 運営団体ログイン
- `OrganizerMember`
  - 運営側の担当者
- `League`
  - WMGP シーズン単位
- `Team`
  - 参加チーム
- `Participant`
  - チーム所属プレイヤー
- `Transfer` または `RosterChange`
  - 追加、削除、移籍履歴
- `Week`
  - 週単位の進行
- `Match`
  - 対戦カード
- `MatchSchedule`
  - 確定日時、ルーム担当、連絡状況
- `MatchOrder`
  - ラウンドごとの出場順
- `DeckSubmission`
  - 提出済み状態と証跡
- `Round`
  - ラウンド単位の結果
- `IndividualMatch`
  - 各プレイヤーのマッチ結果
- `Penalty`
  - 警告、遅延、没収、裁定メモ

## Must-Have Screens

- ログイン
- ダッシュボード
  - 今週の未確定試合
  - 直近締切
  - 要対応の裁定や提出不足
- チーム一覧、詳細
- ロスター変更管理
- 対戦カード一覧
- 試合詳細
  - 日程
  - オーダー
  - デッキ提出
  - ラウンド結果
  - 裁定メモ
- 順位表
- リザルト出力用画面

## Rules That Must Be Enforced

- 1 チーム 6 名以上 15 名以下
- エントリー上限 64 チーム
- メンバー変更は日曜 23:59 締切、翌週反映
- 移籍はチームごとに大会中 2 回まで
- 試合日程は金曜 23:59 までに確定
- オーダー提出は試合 2 時間前まで
- 試合は 3 対 3
- 最大 3 ラウンド、2 ラウンド先取
- 各ラウンド 50 分制限
- 順位はルールブック記載の優先順で算出

## Operational Assumptions

- Discord が一次情報源で、アプリは運営が転記、確定する台帳
- Discord のタイムスタンプ基準を扱えるよう、入力時刻と確定時刻を保持する
- チーム代表者や選手にログインを配らない
- 運営が代理入力することを前提に、入力負荷を最小化する

## Deferred Until After PoC

- チーム側セルフエントリー
- チーム側日程回答
- Discord OAuth、Google OAuth
- Discord 連携による自動通知送信
- 汎用大会エンジン化
- 高度な分析や公開プロフィール

## Recommended Implementation Order

1. チーム、参加者、週、試合の台帳整備
2. 試合詳細画面で日程、オーダー、結果入力を一気通貫にする
3. 順位計算と対戦カード一覧を安定させる
4. リザルト出力と通知用文面生成を載せる
5. その後に担当者記録や管理権限を強化する
